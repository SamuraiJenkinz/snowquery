# Milestone v2.2: SNOWGREP Visual Revamp — Loro Piana Quiet Luxury

**Status:** ✅ SHIPPED 2026-05-24
**Phases:** 6-12 (7 phases)
**Total Plans:** 17

## Overview

Replace the brutalist terminal CSS with an editorial Loro Piana quiet luxury aesthetic across every screen — splash, sidebar, main chat, results, and charts — without touching v2.1 LLM behavior and without losing any v2.1 functionality.

The seven-phase structure followed a strict **foundation → screen restyle → results layer → polish → documentation/gate → doc-accuracy cleanup** order. Phase 6 landed the design tokens, fonts, palette, and global CSS module that every subsequent phase consumed. Phase 7 landed the splash screen (depends on Phase 6 fonts). Phase 8 restyled the sidebar AND main panel together — disjoint DOM regions sharing the same CSS module, executed as two parallelizable waves inside one phase. Phase 9 layered the editorial dataframe + Altair theme onto the main panel. Phase 10 polished edge states (empty state, loading, error, provider warning) across all screens. Phase 11 closed documentation and locked the acceptance gate — including the load-bearing invariant that all 22 v2.1 Phase 5 UI tests stay green throughout. Phase 12 was inserted late to close the three low-severity doc-drift items surfaced by the milestone audit before final archival.

Three Stitch mockups (`.planning/design-mockups/00-splash-helix.png`, `01-main-chat.png`, `02-results-chart.png`) were the visual contract; the `loro-piana-aesthetic` skill was the design system source.

**v2.1 invariants that v2.2 work preserved:**

1. `_render_provenance_caption(provider, model)` MUST NOT read `st.session_state` — AST-based regression test locks this. Phase 8 restyle (MAIN-04) only changed CSS, not the helper's read sources.
2. Locked v2.1 UI strings stayed verbatim: `"LLM provider"` (selectbox label), `"Azure OpenAI"`, `"Anthropic Claude (MGTI)"`, `"QUERY DISABLED — see sidebar warning"`. Phase 5 had 22 tests asserting these; Phase 11 gated v2.2 with them still green.
3. All 91 v2.1 tests stayed green throughout v2.2. Final test count: 103 (91 v2.1 + 12 v2.2 visual regression).

## Milestone Summary

**Decimal Phases:** None. Phase 12 was a late-inserted full phase (not decimal) to close audit doc-drift before final archival.

**Key Decisions:**

- **Phase 8 merged from two phases to one (two waves)** — sidebar + main panel target disjoint DOM regions and consume Phase 6 tokens read-only; mergeable into one phase with two parallel waves (Wave A: SBR-*, Wave B: MAIN-*). Reduced v2.2 phase count from 8 to 7 without losing coverage.
- **Editorial HTML hero + `st.expander` for native dataframe** (DVZ-02) — chosen over fighting glide-data-grid CSS. Removes 4-6h estimate risk; zero functionality loss vs. v2.1.
- **`st.radio(horizontal=True)` for MODE with sage active-dot via `:has(input:checked)`** (SBR-03 approved deviation) — three `st.button` pills failed because sibling DOM structure prevented active-state CSS matching; native primitive with `:has()` selector is cleaner and Streamlit-idiomatic.
- **Branded SNOWGREP logo PNG for both sidebar and main panel hero** (MAIN-01 approved deviation) — single asset at `static/snowgrep-logo.png` served via Streamlit static serving (`enableStaticServing = true`). Dark backdrop intentional — high-contrast brand mark against warm-beige page.
- **DVZ-03 truncation cap at 50 rows, not REQUIREMENTS.md's `>1000`** — editorial table is the hero view and must be scannable at a glance. Encoded as `_TRUNCATION_CAP = 50`.
- **Altair 6 `@alt.theme.register("loro_piana", enable=True)` decorator API** — NOT the deprecated `alt.themes.*` namespace. SNOWGREP canonical pattern for Altair theme registration; applies process-wide at import time.
- **Side-effect import for theme activation** — `import src.ui.altair_theme  # noqa: F401` in `app.py` triggers the decorator at module load. Module is never referenced by name.
- **`src/ui/css.py` is the single source of truth for hex values** — no module is permitted to hardcode Loro Piana hex values or Streamlit selector overrides. Documented exception: `src/chart_generator.py:350` `mark_text(color="#2C2420")` and `mark_line(color="#C0392B")` are mark properties (not color encodings) where theme `range.category` does not apply.
- **`VIBRANT_PALETTE` (`src/ui/altair_theme.py:42-48`) is the canonical chart data palette source** — five hex values: `#C0392B` (crimson), `#2E5BBA` (royal blue), `#2E7D32` (forest green), `#E67E22` (burnt orange), `#F39C12` (mustard yellow).
- **Pure-string-builder pattern for renderers** — `src/ui/results.py` functions return HTML strings; `st.markdown(html, unsafe_allow_html=True)` wrapping happens at call sites in `app.py`. No Streamlit import in renderer module. All dynamic inputs `html.escape()`-wrapped.
- **`PYTHONPATH=. python -m pytest tests/`** pattern documented — repo has no `pyproject.toml` / `pytest.ini` / `conftest.py`. Bare `pytest tests/` fails with `ModuleNotFoundError: No module named 'src'`. Adding `[tool.pytest.ini_options]\npythonpath = ["."]` was out of scope for v2.2; deferred to v2.3 backlog.
- **`.lp-warn-pill` is the v2.2 soft-warning pill primitive** — reused by USING DEFAULT PASSWORD, NO DATA LOADED, and UNLOCK UPLOAD (via per-button `.st-key-unlock_upload` override). Phase 9-10 reused this class for sibling status indicators rather than re-rolling.
- **Phase 12 inserted late to close audit doc-drift** — initial milestone audit returned `tech_debt` status with 4 low-severity doc-drift items in USER_GUIDE.md / README.md. Phase 12 (`doc-accuracy-cleanup`) closed all four; re-audit promoted milestone to `passed`.
