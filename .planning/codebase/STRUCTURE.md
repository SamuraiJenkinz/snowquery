## Directory Purposes

**src/**
- Purpose: Core application logic, all query and data processing modules
- Contains: Ingest, embeddings, routing, SQL generation, semantic search, visualization, utilities
- Key files: `query_router.py` (orchestration hub), `ingest.py` (data entry point), `embeddings.py` (ML)

**config/**
- Purpose: Centralized configuration (via single `config.py` file)
- Contains: Azure OpenAI endpoints, database paths, field definitions, query limits
- Loaded by: All src/ modules and app.py

**db/**
- Purpose: Persistent data storage at runtime
- Contains: `incidents.duckdb` (SQL table), `chroma/` (vector embeddings)
- Generated by: Ingest (DuckDB), embeddings (ChromaDB)
- Not committed: `.gitkeep` placeholder preserves directory structure

**data/**
- Purpose: Staging area for CSV uploads (if ever written to disk; currently uses BytesIO)
- Generated by: Streamlit file_uploader (not currently used)
- Not committed: `.gitkeep` placeholder

**deploy/**
- Purpose: Windows-specific deployment and service management scripts
- Contains: PowerShell scripts for Python build from source, Windows Service management, scheduled task setup
- Used by: Infrastructure/DevOps engineers for production deployment

**.planning/codebase/**
- Purpose: GSD (Good Software Design) planning documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Used by: `/gsd:plan-phase` and `/gsd:execute-phase` commands for context

## Naming Conventions

**Files:**
- Module files: snake_case (e.g., `query_router.py`, `semantic_search.py`)
- Entry points: single names or descriptive (e.g., `app.py`, `config.py`)
- Config: single file `config.py` (all constants defined here)
- Tests: not yet present; would follow `test_*.py` pattern

**Directories:**
- Packages: lowercase, underscores (e.g., `src/`, `db/`, `data/`)
- Deployment: `deploy/` (no underscores)
- Planning: `.planning/` (hidden, dot prefix)
- Documentation: `docs/`

**Python Identifiers:**
- Functions: snake_case (e.g., `classify_intent()`, `query_with_sql()`)
- Classes: PascalCase, used for exceptions (e.g., `QueryError`, `IngestionError`)
- Constants: UPPER_SNAKE_CASE (e.g., `EMBEDDING_MODEL`, `TEXT_FIELDS`, `COLLECTION_NAME`)
- Private functions: leading underscore (e.g., `_heuristic_classify()`, `_detect_column_type()`)

**Modules as Namespaces:**
- `query_router`: All intent classification and routing logic
- `sql_generator`: SQL generation, security checks, execution
- `semantic_search`: Vector search, enrichment, distance conversion
- `ingest`: CSV parsing, schema detection, DuckDB operations
- `embeddings`: Model/client initialization, text prep, batch embedding
- `chart_generator`: Chart type inference, visualization rendering
- `utils`: Logging, exceptions, formatting helpers
- `config`: All configuration (no logic)

## Where to Add New Code

**New Feature (e.g., "Add export to Parquet"):**
- Primary code: `src/ingest.py` (add export_parquet function) or create `src/export.py`
- Tests: `tests/test_export.py` (would create test directory)
- Config updates: `config.py` if new settings needed
- UI integration: `app.py` if user-facing (add button/option in sidebar or results section)

**New Query Mode (e.g., "Add keyword search mode"):**
- Core logic: New function in `src/query_router.py` or create `src/keyword_search.py`
- Intent detection: Add keyword patterns to `classify_intent()` in `src/query_router.py`
- Router update: Add case in `route_query()` to call new search method
- Tests: `tests/test_query_router.py`

**New Visualization Type (e.g., "Add scatter plot"):**
- Chart logic: `src/chart_generator.py` → add to `CHART_PATTERNS` dict, add pattern matching, add chart generation in `generate_chart()`
- Update inference: `infer_chart_type()` to detect scatter requirements and return config
- Dark theme: Add to `configure_chart_theme()` if custom styling needed

**New Data Field Type (e.g., "Add support for duration fields"):**
- Config: Add to `TEXT_FIELDS`, `DATE_FIELDS`, or `CATEGORY_FIELDS` in `config.py`
- Type detection: Update `_detect_column_type()` heuristics in `src/ingest.py`
- Embedding: Update `_prepare_text()` in `src/embeddings.py` if text field
- Schema formatting: Update `format_schema_for_llm()` in `src/utils.py` if LLM context affected

**New External Integration (e.g., "Add Pinecone vector store instead of ChromaDB"):**
- New module: Create `src/vector_store.py` with provider-agnostic interface
- Client init: Add `initialize_vector_store()` function (wraps Pinecone, ChromaDB, etc.)
- Usage: Update `src/embeddings.py` and `src/semantic_search.py` to call interface, not ChromaDB directly
- Config: Add `VECTOR_STORE_PROVIDER` to `config.py` with fallback to ChromaDB

## Special Directories

**db/ (Database Persistence):**
- Purpose: Stores SQL and vector data at runtime
- Generated: Yes, created by app at first data load
- Committed: No (`.gitkeep` only)
- Contents:
  - `incidents.duckdb`: Binary DuckDB file, grows with data
  - `chroma/`: Directory, contains ChromaDB persistent indexes
- Cleanup: Delete these directories to reset data between testing sessions

**data/ (CSV Staging):**
- Purpose: Potential staging for uploaded CSVs (currently app reads into BytesIO)
- Generated: Possibly by future features
- Committed: No (`.gitkeep` only)
- Note: Currently not used; Streamlit uploads stay in memory

**.planning/codebase/ (GSD Documents):**
- Purpose: Architecture and design documentation for `/gsd` commands
- Not generated: Written manually or by Claude Code agents
- Should be committed: Yes, central to GSD workflow
- Contents: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md

**deploy/ (Windows Deployment):**
- Purpose: Scripts and docs for Windows-based deployment
- Not generated: Written by ops engineer
- Should be committed: Yes, for reproducible deployments
- Examples: `build_python_311.ps1` (compile Python from source on Windows), `manage_service.ps1` (Windows Service wrapper)
