## Objective

1. **Add side-effect import** of `src.ui.altair_theme` (activates the loro_piana theme globally) and named imports from `src.ui.results`.

Maps to **DVZ-02** (expander wiring + zero functionality loss), **DVZ-03** (50-row truncation goes live via Plan 02's renderer), **DVZ-05** (chart_feedback restyle + theme activation), and the 0-row editorial empty-state edge case from 09-CONTEXT.md.

```python
def display_results(df: pd.DataFrame, sql: str | None, query_id: str, executive_summary: str | None = None, chart=None, chart_feedback: str | None = None):
    """Display query results with editorial hero + interactive expander.

    Phase 9 contract:
    - 0-row df → editorial NO RESULTS card ONLY (no table, no expander, no chart,
      no chart_feedback warning). Short-circuits before any other render.
    - 1-50 row df → editorial HTML table as hero, expander beneath holds native
      st.dataframe + EXPORT CSV.
    - >50 row df → editorial hero shows df.head(50) + truncation caption;
      expander's st.dataframe receives the FULL df.
    - chart is None AND chart_feedback is set (1-data-point case) → editorial
      CHART UNAVAILABLE restyle replaces the v2.1 st.warning("📊 ...") line.

    The loro_piana Altair theme is active globally via the side-effect import
    of src.ui.altair_theme at module top — st.altair_chart(chart) inherits it
    automatically.
    """
    # Phase 9 DVZ-02 edge: 0-row → editorial empty state ONLY.
    # Suppresses chart_feedback per 09-CONTEXT.md edge states.
    if df.empty:
        st.markdown(_render_empty_state(), unsafe_allow_html=True)
        return

    # Executive summary (unchanged from v2.1)
    if executive_summary:
        st.markdown("### EXECUTIVE SUMMARY")
        st.markdown(f'<div class="query-box">{executive_summary}</div>', unsafe_allow_html=True)
        st.divider()

    # Chart display (theme baked in via altair_theme side-effect import).
    if chart is not None:
        st.markdown("### VISUALIZATION")
        # Show feedback about chart adjustments (e.g., "Switched to bar chart")
        if chart_feedback:
            st.markdown(_render_chart_unavailable(chart_feedback), unsafe_allow_html=True)
        st.altair_chart(chart, use_container_width=True)
        st.divider()
    elif chart_feedback:
        # 1-data-point or other "chart couldn't be generated" case.
        # Phase 9 DVZ-05 edge: editorial restyle, not st.warning.
        st.markdown(_render_chart_unavailable(chart_feedback), unsafe_allow_html=True)

    # Phase 9 DVZ-01 — editorial hero table.
    # Pass RAW df (not format_dataframe_for_display); the renderer applies its
    # own 140-char short_description truncation and 50-row cap. The expander
    # below shows the full df via native st.dataframe.
    st.markdown(_render_editorial_table(df), unsafe_allow_html=True)

    # Phase 9 DVZ-02 — interactive expander beneath the editorial hero.
    # st.expander has NO key= parameter in Streamlit 1.52.1 — no widget-id
    # collision in chat history because the expander itself owns no state.
    with st.expander("EXPAND · INTERACTIVE VIEW", expanded=False):
        st.dataframe(
            df,
            use_container_width=True,
            hide_index=True,
        )

        csv_data = dataframe_to_csv_bytes(df)
        st.download_button(
            label="EXPORT CSV",
            data=csv_data,
            file_name=generate_export_filename("incidents"),
            mime="text/csv",
            key=f"export_{query_id}",
        )

    # SQL expander (unchanged — outside the EXPAND · INTERACTIVE VIEW for
    # visual separation; SQL is a debugging affordance, not data).
    if sql and st.session_state.get("show_sql", True):
        with st.expander("GENERATED SQL"):
            st.code(sql, language="sql")
```

Critical notes on the rewrite:

- **0-row branch is FIRST.** Returns immediately. Skips executive summary, chart, chart_feedback — all of which are misleading or redundant on an empty result.
- **`format_dataframe_for_display(df)` is NO LONGER called.** The editorial renderer truncates `short_description` at 140 chars (richer than utils.py's 100-char default), and the native `st.dataframe` in the expander handles its own column-width concerns (per 09-RESEARCH.md Pitfall 8).
- **The `priority_cols` column-ordering loop is REMOVED** — `_render_editorial_table` does its own column reordering via its internal `_PRIORITY_COLUMN_ORDER`. The expander's `st.dataframe` shows the natural column order from the source df (acceptable; the interactive view is for power-use).
- **`_render_chart_unavailable` replaces BOTH** `st.info(f"📊 {chart_feedback}")` AND `st.warning(f"📊 {chart_feedback}")`. The v2.1 distinction between "info" (chart present + adjustment notice) and "warning" (chart absent + reason) collapses into a single editorial restyle. The label `CHART UNAVAILABLE` is slightly loose in the "Switched to bar chart" sub-case but acceptable — the body text carries the specific message. This is a documented unification.
- **`st.columns([1, 3])` is REMOVED.** v2.1 placed CSV in col1 and SQL expander in col2 side-by-side. Phase 9 flow is vertical: editorial table → EXPAND expander (containing dataframe + CSV) → GENERATED SQL expander beneath. CSV moves INSIDE the EXPAND expander per 09-CONTEXT.md "Inside the expander".
- **`csv_data = dataframe_to_csv_bytes(df)`** is computed inside the `with st.expander(...)` block — `df` is in scope.
- **`st.download_button` keeps `key=f"export_{query_id}"`** to avoid `DuplicateWidgetID` errors in chat history (Pitfall 4).
- **`st.dataframe` is called WITHOUT `key=`** — selection is not enabled, no widget state, no collision risk (Pitfall 4).
- The function signature is unchanged — same name, same parameter list, same defaults. Callers in `render_chat_history` and `render_main_content` are call-compatible.
