# Phase 9, Plan 03 — Chart generator restyle (horizontal bar, vibrant palette, theme consumption)

## Objective

Restyle `src/chart_generator.py::generate_chart` for Phase 9 DVZ-05. Three concrete changes:

1. **Delete** the v2.1 dark-theme machinery: `CHART_COLORS` constant (lines 17-28) and `configure_chart_theme()` function (lines 261-295). Delete the single `chart = configure_chart_theme(chart)` call (line 424). The new `loro_piana` theme (Plan 01) handles all chrome.
2. **Rewrite the bar chart branch** for HORIZONTAL orientation: encoding flips from `x=category, y=value, sort='-y'` to `x=value (quantitative), y=category (nominal), sort='-x'`. Add a layered `mark_text` showing value labels to the right of each bar (Inter 12px charcoal `#2C2420`, comma-formatted). Returns `alt.LayerChart` from `alt.layer(bars, labels)`.
3. **Replace** the two surviving `CHART_COLORS` references at lines 343 and 378 with `VIBRANT_PALETTE` imported from `src.ui.altair_theme`. Replace `CHART_COLORS[0]` at line 397 (line chart mark color) with the literal `"#C0392B"` (crimson, matches `VIBRANT_PALETTE[0]`).

Wave 2, depends on Plan 01 (this plan imports `VIBRANT_PALETTE` from `src.ui.altair_theme`).

Logic for chart inference (`infer_chart_type`), category consolidation (`_consolidate_small_categories`), and `chart_feedback` strings is UNCHANGED. Constants `MAX_PIE_SLICES = 10`, `MAX_BAR_CATEGORIES = 20`, `MIN_ROWS_FOR_CHART = 2` are PRESERVED.

- Single-series charts have no legend per 09-CONTEXT.md "Legend rules". Bar charts get `legend=None` on the color encoding (y-axis tick labels already name categories); pie charts retain a legend; line charts have no color encoding, so no legend by default.

**Rewrite bar branch** (formerly lines 356-389): FULL REWRITE. Replace the entire `elif chart_type == "bar":` block with:

```python
        elif chart_type == "bar":
            # Consolidate categories if needed
            if chart_df[x_col].nunique() > MAX_BAR_CATEGORIES:
                chart_df = _consolidate_small_categories(
                    chart_df, x_col, y_col, MAX_BAR_CATEGORIES
                )

            # Phase 9 DVZ-05: HORIZONTAL bar — value (Q) on x, category (N) on y.
            # sort='-x' on the Y encoding produces "largest bar at top" ordering.
            # See 09-RESEARCH.md Pitfall 5 for the -x vs -y sort gotcha.
            base = alt.Chart(chart_df)

            bars = base.mark_bar().encode(
                x=alt.X(
                    field=y_col,
                    type="quantitative",
                    title=y_col.replace("_", " ").title()
                ),
                y=alt.Y(
                    field=x_col,
                    type="nominal",
                    sort="-x",
                    title=x_col.replace("_", " ").title()
                ),
                color=alt.Color(
                    field=x_col,
                    type="nominal",
                    scale=alt.Scale(range=VIBRANT_PALETTE),
                    legend=None
                ),
                tooltip=[
                    alt.Tooltip(x_col, title=x_col.replace("_", " ").title()),
                    alt.Tooltip(y_col, title=y_col.replace("_", " ").title())
                ]
            )

            # Layered text marks — value labels to the right of each bar.
            # Inter 12px charcoal, comma-formatted; matches mockup
            # .planning/design-mockups/02-results-chart.png.
            labels = base.mark_text(
                align="left",
                baseline="middle",
                dx=4,
                font="Inter",
                fontSize=12,
                color="#2C2420"
            ).encode(
                x=alt.X(field=y_col, type="quantitative"),
                y=alt.Y(field=x_col, type="nominal", sort="-x"),
                text=alt.Text(field=y_col, type="quantitative", format=",")
            )

            chart = alt.layer(bars, labels).properties(
                width=600,
                height=max(200, len(chart_df) * 32),
                title=f"{y_col.replace('_', ' ').title()} by {x_col.replace('_', ' ').title()}"
            )
```

Key bar rewrite points:
- `x` is now the QUANTITATIVE value (formerly `y`); `y` is now the NOMINAL category (formerly `x`).
- `sort="-x"` on the Y encoding sorts categories DESCENDING by the X-axis value — largest at top. Horizontal analog of the old `sort="-y"`.
- The layered `mark_text` shares `x` + `y` encodings with the bars, adds `text` encoding with `format=","` for thousand-separator formatting.
- `alt.layer(bars, labels)` returns `alt.LayerChart` — `st.altair_chart()` accepts it (verified per 09-RESEARCH.md Open Question 4).
- `height=max(200, len(chart_df) * 32)` scales the chart height to the number of bars (32px each, 200px floor). The old fixed `height=400` would crowd many-bar charts in horizontal layout.
- `legend=None` on the color encoding — y-axis tick labels already name categories (09-CONTEXT.md).
