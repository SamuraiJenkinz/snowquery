## Decisions Made

1. **str.format() requires doubled CSS braces** ‚Äî The plan incorrectly stated "CSS braces remain single ‚Äî they pass through .format() untouched." This is false. Python's `str.format()` parses ALL `{...}` as placeholders regardless of content. All CSS rule blocks, keyframe percent blocks, and JS function bodies required `{{` and `}}` escaping. First write attempt raised `KeyError: '\n      --lp-bg'` (the entire `:root { ... }` block was parsed as one placeholder). Fixed by doubling all non-placeholder braces.

2. **JS const DISMISS_TYPE pattern (plan option b)** ‚Äî Avoids `{type: 'snowgrep-splash-dismiss'}` object literal shape in the template which would cause `KeyError: 'type'` at format time. `const DISMISS_TYPE = 'snowgrep-splash-dismiss';` declared once; comparison uses `ev.data.type !== DISMISS_TYPE`.

3. **Comment hex quotes removed** ‚Äî Python comments like `# "#F5F0EB"` with quoted hex strings still match the brand-hex-literal grep test. Replaced with descriptive comments (`# warm off-white`) to pass the zero-literal invariant cleanly.

4. **`import streamlit.components.v1 as components` ‚Üí `components.html()`** ‚Äî The import alias binds `components` to the v1 submodule directly, so the call is `components.html(...)` not `components.v1.html(...)`. Both the `inspect.getsource` check and monkey-patch test account for this pattern.
